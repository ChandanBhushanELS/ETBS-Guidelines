Solution Template

A Simple Example

Here is a simple example using our template:

<article>
    <header>
        <meta name="solution_id" content="12345">
        <meta name="question_code" content="0010A2">
        <address>
            <span property="author_email">chandan.kumar@evelynlearning.com</span>
            <time property="date_created">2019-02-24</time>
        </address>
    </header>
    <main>
        <section class="answer" data-type="number">
            31
        </section>
        <section class="step_by_step">
            <div data-type="step" data-id="1">
                <span class="left" data-type="text">
                    Order of operations says that we apply multiplication first.
                </span>
                <span class="right" data-type="formula">
                    $$= 4 + 30 - 3$$
                </span>
            </div>
            <div data-type="step" data-id="2">
                <span class="left" data-type="text">
                    Order of operations says that addition and subtration are applied left to right. This gives our final answer.
                </span>
                <span class="right" data-type="formula">
                    $$= 31$$
                </span>
            </div>
        </section>
    </main>
</article>

Let's go through this example line by line to define the full template.

Basic Structure

Every solution has an <article> tag as its outermost parent. The article tag has no attributes.

<article>
    ...
</article>

Header

The header portion of a solution defines necessary metadata to identify the solution and is surrounded by <header> tags. None of this information is publically displayed.

<header>
    <meta name="solution_id" content="12345">
    <meta name="question_code" content="0010A2">
    <address>
        <span property="author_email">chandan.kumar@evelynlearning.com</span>
        <time property="date_created">2019-02-24</time>
    </address>
</header>

First, we have two <meta> tags. These tags have a name attribute which states the name of the metadata and a content attribute which states the value. We will always have the following two <meta> tags:

  1. solution_id - the solution_id for the solution your are authoring, found in your assignment list.
  2. question_code - the question_code for the question you are referencing while authoring, found in your assignment list.

Next we have the author information. This information is surrounding by <address> tags. Within the block, there is a <span> tag with a property attribute set to "author_email". This tag should contain your email address and is how we identify who wrote a solution. Always use the same email while authoring.

The next tag in the <address> block is a <time> tag with a property attribute of "date_created". This tag should contain the date you started authoring the solution, in YYYY-MM-DD format. This is the only acceptable format for the "date_created" value.

Solution Body

The body of the solution is contained in the <main> block. This is the content that will be displayed to our students.

<main>
    ...
</main>

Within the <main> block, parts of the solution will be separated into <section> blocks. Each section corresponds to a component of the solution, such as the final answer, a video, or a step by step explanation. The <section> tag's class specifies what type of component it is and the full component is contained inside. The specific combination of section blocks for each question type is described in Question Types.

Section Classes:

  • answer - contains the final answer(s) for the question
  • explanation - contains the explanation for the correct answer option or solution
  • wrong_explanation - contains the explanation for the incorrect answer option(s) for MCQ or MS questions
  • step_by_step - contains the step-by-step explanation for the question
  • sample_response - contains a sample response for the question (used when a single correct solution is not applicable)

Answer Section

<section class="answer" data-type="number">
    31
</section>

The answer section requires a "data-type" attribute specifying the type of content contained within. See below for available data types.

Explanation Section

<section class="explanation" data-type="text">
    This is some explanation for the correct answer option.
</section>

The explanation section doesn't exist in our example, but is very similar to the answer section. It requires a "data-type" attribute specifying the type of content contained within. See below for available data types.

Wrong Explanation Section

<section class="wrong_explanation" data-type="text" data-answer="A">
    This is some explanation as to why answer option A is incorrect for an MCQ or MS question.
</section>

The "wrong_explanation" section doesn't exist in our example, but is very similar to the answer section. It requires a "data-type" attribute specifying the type of content contained within. See below for available data types.

Step By Step Section

<section class="step_by_step">
    <div data-type="step" data-id="1">
        <span class="left" data-type="text">
            Order of operations says that we apply multiplication first.
        </span>
        <span class="right" data-type="formula">
            $$= 4 + 30 - 3$$
        </span>
    </div>
    <div data-type="step" data-id="2">
        <span class="left" data-type="text">
            Order of operations says that addition and subtration are applied left to right. This gives our final answer.
        </span>
        <span class="right" data-type="formula">
            $$= 31$$
        </span>
    </div>
</section>

The Step By Step section is a wrapper for multiple <div> blocks, each containing a step. Steps may optionally be grouped into larger <div> blocks representing question parts.

A step div looks like this:

<div data-type="step" data-id="1">
    ...
</div>

The "data-type" attribute is always set to "step" and the "data-id" attribute is set to the number of the step. Always start numbering steps at 1 and count up by whole numbers. Never skip a number or use a fractional number (such as 1.5).

The Step Block

<div data-type="step" data-id="2">
    <span class="left" data-type="text">
        Order of operations says that addition and subtration are applied left to right. This gives our final answer.
    </span>
    <span class="right" data-type="formula">
        $$= 31$$
    </span>
</div>

The step block contains the content of a step in your explanation. Steps can either contain a single chunk of content (in which case you should add a "data-type" attribute to the step <div> tag) or a two column layout. Two column layouts should be used as often as possible and consist of two <span> blocks with a class of "left" or "right". You must have exactly one <span> with a left class and one with a right class. You will also need a "data-type" attribute for each <span>.

Questions with Multiple Parts

A part div looks like this:

<div data-type="part" data-id="a">
    <div data-type="step" data-id="1">
        ...
    </div>
</div>

The "data-type" attribute is always set to "part" and the "data-id" attribute is set to the part identifier, as found in your assignment.

Data Types

If no data type is provided, the default is "text". If multiple data types are present in a section (e.g, an image or formula in addition to text), simply assign "text" as the data type attribute.

  • number - the content of the block is a single number (includes dollar amounts, fractions, irrational numbers like pi, etc.)
  • formula - the content of the block is a single katex formula. See the Formula Guidelines for more details.
  • image - the content of the block is an image. See the Image Guidelines for more details.
  • graph - the content of the block is a graph. See the Graph Guidelines for more details.
  • video - the content of the block is a video
  • mcq_option - the content of the block is an answer identifier (a, b, etc.) from an MCQ or MS question.
  • boolean - the content of the block is true or false.
  • text - the content of the block is plain text (there shouldn't be another data type that fits better)
