Question Types

This page provides instructions for authoring HTML solutions using different question type templates. For instructions on how to complete the <header> block, see the Solution Template page.

MCQ | MS | TCQ (TF)| FIB | SA | ES | CALC | ORD | GR | DRW | LDGR | MAT | CL

Multiple Choice (MCQ)

A multiple choice question is a problem that has a prompt and provides several possible discrete answer options where only 1 of the provided answer options is the correct answer.

Example:

Which of the following is a type of plant?
A) fern
B) canine
C) mushroom
D) shark

There is one and only one correct response to this question: A) fern

Directions for authoring MCQ solutions:

There are always three types of <section> blocks present in an MCQ solution:

  • answer
  • explanation
  • wrong_explanation
<article>
    <header>
        ...
    </header>
    <main>
        <section class="answer" data-type="mcq_option">
            A
        </section>
        <section class="explanation" data-type="text" data-answer="A">
            [Explanation for A]
        </section>
        <section class="wrong_explanation" data-type="text" data-answer="B">
            [Explanation for B]
        </section>
        <section class="wrong_explanation" data-type="text" data-answer="C">
            [Explanation for C]
        </section>
        <section class="wrong_explanation" data-type="text" data-answer="D">
            [Explanation for D]
        </section>
    </main>
</article>

For the first <section> block, the solution is provided and the letter of the correct answer option is provided in the format that matches the textbook (uppercase vs. lowercase). The attributes for the section are always class="answer" and data-type="mcq_option".

Example: 

<section class="answer" data-type="mcq_option">
    A
</section>

For the second <section> block, the explanation for the correct answer option is provided to the student. The attributes for the section are class="explanation" and data-type="text". The data-answer attribute is always set to be the letter of the correct answer option. [Note: it is possible to assign a data-type attribute other than "text" in some instances. See the Solution Template page for a list of possible data types.]

Example: 

<section class="explanation" data-type="text" data-answer="A">
    Provide an explanation for why this is the correct answer. The explanation should be relevant
    to the overall question and help a student better understand the concept being tested. Do not include the text
    of the stem or the answer option (as it appears in the textbook) in your solution.
</section>

For the third and all subsequent <section> blocks, the explanations for the incorrect answer options are provided to the student. There will be a wrong_explanation section block provided for each incorrect answer option. The attributes for the section are class="wrong_explanation" and data-type="text". The data-answer attribute is set to be the letter of the incorrect answer option for that section. [Note: it is possible to assign a data-type attribute other than "text" in some instances. See the Solution Template page for a list of possible data types.]

Example: 

<section class="wrong_explanation" data-type="text" data-answer="B">
    Provide an explanation for why answer option B is an incorrect answer to the question.
    The explanation should be relevant to the overall question and help a student better understand
    the concept being tested. Do not include the text of the stem or the answer option (as it appears
    in the textbook) in your solution.
</section>

The following are key features of a quality explanation:

  • supports a student's understanding of the overall concept being assessed (avoids superficial explanations)
  • is specific and concise
  • uses precise subject matter terminology
  • is 100% accurate
  • is clear of grammar and mechanical issues
  • targets the appropriate audience in terms of student-friendly language and appropriate content
  • is original work in the SME’s own words
  • does not cue the correct response (for incorrect answer option explanations)

Multiple Select (MS)

A multiple select question is a problem that has a prompt and clearly provides several possible answer options where more than 1 of the answer options could be a correct answer. Note: some MS questions can have a single answer option keyed, but the question prompt allows for more than one answer option to be selected (e.g., "Select all the apply.").

Example:

Which of the following are plants? Select ALL that apply.
A) fern
B) canine
C) mushroom
D) tree

There are two correct answer options that should both be keyed as part of the correct response: Option (A) and Option (D).

Directions for authoring MS solutions:

There are always three types of <section> blocks present in an MS solution:

  • answer
  • explanation
  • wrong_explanation

The main difference between an MCQ and an MS solution is that an MS solution can have multiple class="answer" and class="explanation" sections.

<article>
    <header>
        ...
    </header>
    <main>
        <section class="answer" data-type="mcq_option">
            A
        </section>
        <section class="answer" data-type="mcq_option">
            D
        </section>
        <section class="explanation" data-type="text" data-answer="A">
            [Explanation for A]
        </section>
        <section class="explanation" data-type="text" data-answer="D">
            [Explanation for D]
        </section>
        <section class="wrong_explanation" data-type="text" data-answer="B">
            [Explanation for B]
        </section>
        <section class="wrong_explanation" data-type="text" data-answer="C">
            [Explanation for C]
        </section>
    </main>
</article>

For an MS question, there can be multiple answer <section> blocks depending on how many answer options are correct. Each answer <section> block includes the letter of a correct answer option (provided in the format that matches the textbook, uppercase vs. lowercase). The attributes for each answer section block are always class="answer" and data-type="mcq_option".

Example: 

<section class="answer" data-type="mcq_option">
    A
</section>
<section class="answer" data-type="mcq_option">
    D
</section>

For the next <section> blocks, an explanation for each correct answer option is provided to the student. The attributes for the section are class="explanation" and data-type="text". The data-answer attribute is always set to be the letter of the correct answer option. [Note: it is possible to assign a data-type attribute other than "text" in some instances. See the Solution Template page for a list of possible data types.]

Example: 

<section class="explanation" data-type="text" data-answer="A">
    Provide an explanation for why this is the correct answer. The explanation should be relevant
    to the overall question and help a student better understand the concept being tested. Do not include the text
    of the stem or the answer option (as it appears in the textbook) in your solution.
</section>
<section class="explanation" data-type="text" data-answer="D">
    This is an explanation for the second correct answer option: option (D).
</section>

For the final <section> blocks, the explanations for the incorrect answer options are provided to the student. There will be a wrong_explanation section block provided for each incorrect answer option. The attributes for the section are class="wrong_explanation" and data-type="text". The data-answer attribute is set to be the letter of the incorrect answer option for that section. [Note: it is possible to assign a data-type attribute other than "text" in some instances. See the Solution Template page for a list of possible data types.]

Example: 

<section class="wrong_explanation" data-type="text" data-answer="B">
    Provide an explanation for why answer option B is an incorrect answer to the question.
    The explanation should be relevant to the overall question and help a student better understand
    the concept being tested. Do not include the text of the stem or the answer option (as it appears
    in the textbook) in your solution.
</section>
<section class="wrong_explanation" data-type="text" data-answer="C">
    Provide an explanation for why answer option C is an incorrect answer to the question.
    The explanation should be relevant to the overall question and help a student better understand
    the concept being tested. Do not include the text of the stem or the answer option (as it appears
    in the textbook) in your solution.
</section>

The following are key features of a quality explanation:

  • supports a student's understanding of the overall concept being assessed (avoids superficial explanations)
  • is specific and concise
  • uses precise subject matter terminology
  • is 100% accurate
  • is clear of grammar and mechanical issues
  • targets the appropriate audience in terms of student-friendly language and appropriate content
  • is original work in the SME’s own words
  • does not cue the correct response (for incorrect answer option explanations)

Two Choice (TCQ)

A two choice question is a problem that has a prompt and provides two possible discrete answer options where only 1 of the provided answer options is the correct answer. This question type should be used for TF.

Non-TF Example:

For the following cost, determine whether it is a fixed or variable cost:
Purchase of a new truck

There is one and only one correct response for this question:
fixed

Directions for authoring TCQ solutions:

There are three types of <section> blocks present in a non-TF TCQ solution:

  • answer
  • wrong_answer
  • explanation
<article>
    <header>
        ...
    </header>
    <main>
        <section class="answer" data-type="text">
            fixed
        </section>
        <section class="wrong_answer" data-type="text">
            variable
        </section>
        <section class="explanation" data-type="text">
            Variable costs are costs that vary with output, while fixed costs do not change regardless of the level of output.
            The cost of purchasing a truck does not change regardless of the number of trips it makes, so the cost is fixed.
        </section>
    </main>
</article>

For the first <section> block, the correct answer is provided, this should generally be no longer than a couple words. The attributes for the section are always class="answer" and data-type="text".

Example: 

<section class="answer" data-type="text">
    fixed
</section>

For the second <section> block, the incorrect answer option is provided to the student, similar to the previous section. The attributes for the section are class="wrong_answer" and data-type="text".

Example: 

<section class="wrong_answer" data-type="text">
    variable
</section>

For the third <section> block, the explanation for the correct & incorrect answer options is provided to the student. The attributes for the section are class="explanation" and data-type="text".

Example: 

<section class="wrong_explanation" data-type="text">
    Variable costs are costs that vary with output, while fixed costs do not change regardless of the level of output.
    The cost of purchasing a truck does not change regardless of the number of trips it makes, so the cost is fixed.
</section>

The following are key features of a quality explanation:

  • supports a student's understanding of the overall concept being assessed (avoids superficial explanations)
  • is specific and concise
  • uses precise subject matter terminology
  • is 100% accurate
  • is clear of grammar and mechanical issues
  • targets the appropriate audience in terms of student-friendly language and appropriate content
  • is original work in the SME’s own words

True or False (TF) Example:

A True or False TCQ question is a problem for which the correct answer can only be “true” or “false.”

Is this statement true or false? A fern is a type of plant.

The correct response is "true."

Directions for authoring TF solutions:

There are always two types of <section> blocks present in a TF solution:

  • answer
  • explanation
<article>
    <header>
        ...
    </header>
    <main>
        <section class="answer" data-type="boolean">
            true/false
        </section>
        <section class="explanation" data-type="text">
            [Explanation]
        </section>
    </main>
</article>

For a TF question, there is a single answer <section>. The attributes for the answer section block are always class="answer" and data-type="boolean". The answer can only be "true" or "false".

Example: 

<section class="answer" data-type="boolean">
    true
</section>

For the second <section>, an explanation for the answer (true or false) is provided to the student. The attributes for the section are class="explanation" and data-type="text". 

<section class="explanation" data-type="text">
    Provide an explanation for why the answer is either true or false. The explanation should be relevant
    to the overall question and help a student better understand the concept being tested. Do not include the text
    of the stem (as it appears in the textbook) in your solution.
</section>

Fill-in-the-Blank (FIB)

A Fill-in-the-Blank problem is a statement that includes a blank space (or spaces) for the answer. The statement only makes sense if the correct answer(s) is filled in the provided blank(s). Answer choices may be selected from a given set, or not given at all.

Example:

The capital of California is _____________ and the capital of Colorado is _____________.

The correct response is Sacramento and Denver.

Directions for authoring FIB solutions:

There are always two types of <section> blocks present in a FIB solution:

  • answer
  • explanation
<article>
    <header>
        ...
    </header>
    <main>
        <section class="answer" data-type="text" data-answer="1">[answer for first blank]</section>
        <section class="answer" data-type="text" data-answer="2">[answer for second blank]</section>
        <section class="explanation" data-type="text" data-answer="1">
            [Explanation for blank 1]
        </section>
        <section class="explanation" data-type="text" data-answer="2">
            [Explanation for blank 2]
        </section>
    </main>
</article>

For a FIB question, there can be multiple answer <section> blocks depending on how many blanks are present in the question stem. An answer <section> is provided for each blank and numbered sequentially. The attributes for each answer section block are always class="answer", data-type="text", and data-answer="#" numbered sequentially to correspond with the blanks.

Example: 

<section class="answer" data-type="text" data-answer="1">[answer for first blank]</section>
<section class="answer" data-type="text" data-answer="2">[answer for second blank]</section>

For the next <section> blocks, an explanation for each blank is provided to the student. The attributes for the section are class="explanation" and data-type="text". The data-answer attribute matches the sequence number assigned to that blank in the answer sections.

Example: 

<section class="explanation" data-type="text" data-answer="1">
    A separate explanation is provided for each answer (e.g., for each blank). The explanation and included content
    should be relevant to the overall question and help a student better understand the concept being tested.
</section>
<section class="explanation" data-type="text" data-answer="2">
    A separate explanation is provided for each answer (e.g., for each blank). The explanation and included content
    should be relevant to the overall question and help a student better understand the concept being tested.
</section>

Short Answer (SA)

A Short Answer question is a problem for which an answer can be provided by writing a short response that may or may not require full sentences. SA questions may require answers in the form of definitions, names/lists, explanations, comparisons between ideas that expect a short response, or a step-by-step process. Examples of answers could be the name of a person or the date of a historical event, yes or no, or a single paragraph that contains an explanation for a specific topic. Responses should be less than 100 words.

Example:

What are some of the major differences between the structure of plant and animal cells?

Plant and animal cells differ in some of the following ways:

  • Chloroplasts are present in plants and some algae, but are not found in animal cells. Photosynthesis takes place in chloroplasts.
  • Cell walls are present in plants, but not in animal cells.
  • Vacuoles are membrane-bound sacs that function in storage and transport. A large central vacuole is found in plant cells, but not in animal cells. (Animal cells contain many smaller vacuoles.)
  • Centrioles are found in centrosomes and are present in animal cells, but are absent from plant cells.
  • Lysosomes are present in animal cells, but are rarely seen in plant cells. Lysosomes help digest cellular macromolecules.
  • Cilia are usually observed in animal cells, but are absent from plant cells. Cilia are microtubules that help the cell move.

Directions for authoring SA solutions:

When there is a single correct answer possible, the SA solution should include two types of <section> blocks:

  • answer
  • explanation
<article>
    <header>
        ...
    </header>
    <main>
        <section class="answer" data-type="text">
            [answer text]
        </section>
        <section class="explanation" data-type="text">
            [explanation]
        </section>
    </main>
</article>

When the SA question is more open-ended and more than one correct response is possible, the SA solution will normally include two types of <section> blocks:

  • sample response (required)
  • explanation (optional)
<article>
    <header>
        ...
    </header>
    <main>
        <section class="sample_response" data-type="text">
            [sample response text]
        </section>
        <section class="explanation" data-type="text">
            [explanation or guidance]
        </section>
    </main>
</article>

The answer, sample_response, or explanation sections can be formatted with bulleted lists or html tables to make the content more concise and easier to parse.

<section class="sample_response" data-type="text">
    <ul>
        <li>First point of the sample response.</li>
        <li>Second point of the sample response.</li>
        <li>Third point of the sample response.</li>
    </ul>
</section>
<section class="explanation" data-type="text">
    <p>
        The explanation can focus on a rationale for the correct answer or provide guidance to the student on what should
        be included in their answer (for open-ended questions).
    </p>
    <ul>
        <li>First point of guidance.</li>
        <li>Second point of guidance.</li>
        <li>Third point of guidance.</li>
    </ul>
</section>

Essay (ES)

An essay question is a problem for which an answer can be provided by writing a more in-depth response that requires full sentences. The question may provide a prompt with a situation or topic, and should include instructions for what the essay should include (e.g, specifying a minimum word count or number of pages, requiring a summary/introduction, or specifying the format of the essay [double-spaced, APA, etc.]). The answer would typically include full sentences and consist of one or more paragraphs. Responses should be more than 100 words.

Example:

How has the discovery of DNA been important to society?

Directions for authoring ES solutions:

The ES solution will include two types of <section> blocks:

  • sample_response
  • explanation
<article>
    <header>
        ...
    </header>
    <main>
        <section class="sample_response" data-type="text">
            <p>first paragraph of sample response text</p>
            <p>second paragraph of sample response text</p>
            <p>third paragraph of sample response text</p>
        </section>
        <section class="explanation" data-type="text">
            [explanation and guidance]
        </section>
    </main>
</article>

The explanation section can be formatted with a bulleted list to make the content (guidelines on how to construct the essay) more concise and easier to parse.

<section class="explanation" data-type="text">
    <p>
        The explanation should focus on providing guidance to the student on what should
        be included in their essay.
    </p>
    <ul>
        <li>First point of guidance</li>
        <li>Second point of guidance</li>
        <li>Third point of guidance</li>
    </ul>
</section>

Calculation (CALC)

A question or problem that requires computation to solve. It may instruct the student to solve or prove it. CALC problems often include equations in the questions and/or words such as “Solve for x,” “Calculate x,” “Find x,” “Prove that x…”, “How many…” etc. The answers could be a number, a rate, or an equation that proves a math equation is true. CALC questions will usually require step-by-step solutions.

Example:

Find the value of each permutation: P(6,2)

Directions for authoring CALC solutions:

The CALC solution will normally include two types of <section> blocks:

  • answer
  • step_by_step

When a CALC question is more open-ended and more than one correct response is possible, the CALC solution will include two types of <section> blocks:

  • sample response
  • step_by_step
<article>
    <header>
        ...
    </header>
    <main>
        <section class="answer" data-type="number">
            #
        </section>
        <section class="step_by_step">
            <div data-type="step" data-id="1">
               <span class="left" data-type="text">
                   XXX
               </span>
               <span class="right" data-type="formula">
                   XXX
               </span>
            </div>
            <div data-type="step" data-id="2">
               <span class="left" data-type="text">
                   XXX
               </span>
               <span class="right" data-type="formula">
                   XXX
               </span>
            </div>
        </section>
    </main>
</article>

The first section, the answer or sample_response section, will include the numeric answer for the question. The attributes for this section block are class="answer" or class="sample_response", and data-type="number". Note: the data-type attribute can also be set to data-type="text" if text needs to be included in addition to the numeric answer (e.g., units, descriptions).

<section class="answer" data-type="number">
        #
</section>

The second section, the step_by_step section, provides clear step-by-step instructions directing students how to solve the problem to arrive at the answer. While most CALC questions will incude a step-by-step solution, it's acceptable in certain cases to include an explanation section instead of a step_by_step section. This is reserved for cases where a solution does not warrant a step-by-step template (e.g., does not require a true calculation or requires a single step and doesn't warrant a step-by-step template). 

<section class="step_by_step">
    <div data-type="step" data-id="1">
       <span class="left" data-type="text">
           XXX
       </span>
       <span class="right" data-type="formula">
           XXX
       </span>
    </div>
    <div data-type="step" data-id="2">
       <span class="left" data-type="text">
           XXX
       </span>
       <span class="right" data-type="formula">
           XXX
       </span>
    </div>
</section>

The following are key features of a quality step-by-step explanation:

  • steps should be broken down into logical chunks that the student would be able to understand even if they struggle to understand the entire problem
  • steps should focus on including conceptual-level steps and not separate steps for each simplification of a math formula
  • the first step should outline the conceptual approach to solving the problem (and will serve as the first hint a student receives)
  • each step should include a brief description of what's occurring in that step in the left column and the actual computation ("work") for the step in the right column
  • content provided in the left column will consist primarily of text since it will be a description of what is occurring in the step. The description of the step should explain why this is the logical next step in the process and any pertinent information (vocabulary terms, basic formulas, etc.) that is required to show the work in that step.
  • content provided in the right column will consist of a mix of formulas (formatted as KaTeX) and text. Any formulas needed to solve the step should be shown both as a reference and with the variables for the problem plugged in.
  • the explanation should not exceed eight steps total for any CALC solution (to prevent exceeding eight steps, several steps may need to be grouped together in a logical way)

Ordering (ORD)

A question for which the answer requires the ranking or sequencing of several items provided by the prompt. These questions commonly include the words “Rank” or “Order” in the question and ask for a ranking of the items based on a specific criteria.

Example:

Place the following measurements in order from shortest to longest: 1 m, 10 cm, 1,000 in, 2 ft

The correct order is: 3,1,4,2

Directions for authoring ORD solutions:

There are always two types of <section> blocks present in an ORD solution:

  • answer
  • explanation
<article>
    <header>
        ...
    </header>
    <main>
        <section class="answer" data-type="number" data-answer="1">[order for option 1]</section>
        <section class="answer" data-type="number" data-answer="2">[order for option 2]</section>
        <section class="answer" data-type="number" data-answer="3">[order for option 3]</section>
        <section class="answer" data-type="number" data-answer="4">[order for option 4]</section>
        <section class="explanation" data-type="text" data-answer="1">
            XXX
        </section>
        <section class="explanation" data-type="text" data-answer="2">
            XXX
        </section>
        <section class="explanation" data-type="text" data-answer="3">
            XXX
        </section>
        <section class="explanation" data-type="text" data-answer="4">
            XXX
        </section>
    </main>
</article>

For an ORD question, there will be multiple answer <section> blocks. The attributes for the answer section blocks are class="answer", data-type="number" or "text", and data-answer="#". The data-answer corresponds to the given options and reflects how the options are provided in the question stem. The solution (correct order) for each option is entered as a number.

Example: 

<section class="answer" data-type="number" data-answer="1">3</section>
<section class="answer" data-type="number" data-answer="2">1</section>
<section class="answer" data-type="number" data-answer="3">4</section>
<section class="answer" data-type="number" data-answer="4">2</section>

For the next <section> blocks, an explanation for each option is provided to the student in separate section blocks. The attributes for the sections are class="explanation", data-type="text", and data-answer. The data-answer attribute matches the original sequence number of the options shown in the question stem (e.g., data-answer="1" refers to the first option in the list of possible options given in the question stem).

Example: 

<section class="explanation" data-type="text" data-answer="1">
    A separate explanation is provided for each option. The explanation and included content
    should be relevant to the ordering and help a student better understand the concept being tested.
</section>
<section class="explanation" data-type="text" data-answer="2">
    This would be a separate explanation for the second option.
</section>
<section class="explanation" data-type="text" data-answer="3">
    This would be a separate explanation for the third option.
</section>
<section class="explanation" data-type="text" data-answer="4">
    This would be a separate explanation for the fourth option.
</section>

Graphing (GR)

A question for which the answer must be a graph. Examples includes business questions that ask for a line or curve to be drawn on a provided graph or a graph of a function. Questions will sometimes include the word “Graph” or “Label.” Note that circle graphs (pie charts) or bar graphs should be treated as DRW questions.

Example:

Graph $y=2x+1$

Directions for authoring GR solutions:

The GR solution should include two types of <section> blocks:

  • answer
  • step_by_step

When a GR question is more open-ended and more than one correct response is possible, the GR solution will include two types of <section> blocks:

  • sample_response
  • step_by_step
<article>
    <header>
        ...
    </header>
    <main>
        <section class="answer" data-type="graph">
            <script>
                var board = JXG.JSXGraph.initBoard('jxgboxa', {
                    axis: true,
                    showCopyright: false,
                    showNavigation: false,
                    registerEvents: false
                });
                board.setBoundingBox([-3.5, 0.5, 3.5, -0.1]);
                board.create('functiongraph', [function(x){ return X}]);
            </script>
        </section>
        <section class="step_by_step">
            <div data-type="step" data-id="1">
               <span class="left" data-type="text">
                   XXX
               </span>
               <span class="right" data-type="formula">
                   XXX
               </span>
            </div>
            <div data-type="step" data-id="2">
               <span class="left" data-type="text">
                   XXX
               </span>
               <span class="right" data-type="formula">
                   XXX
               </span>
            </div>
        </section>
    </main>
</article>

The first section, the answer or sample_response section, will include the script for the JSXGraph. The attributes for this section block are class="answer" or class="sample_response", and data-type="graph".

<section class="answer" data-type="graph">
    <script>
        var board = JXG.JSXGraph.initBoard('jxgboxa', {
            axis: true,
            showCopyright: false,
            showNavigation: false,
            registerEvents: false
        });
        board.setBoundingBox([-3.5, 0.5, 3.5, -0.1]);
        board.create('functiongraph', [function(x){ return X}]);
    </script>
</section>

The second section, the step_by_step section, provides clear step-by-step instructions directing students how to develop, label, and format their graphs to fully satisfy the requirements of the question. While most GR questions will incude a step-by-step solution, it's acceptable in certain cases to include an explanation section instead of a step_by_step section. This is reserved for cases where a solution does not warrant a step-by-step template. 

<section class="step_by_step">
    <div data-type="step" data-id="1">
       <span class="left" data-type="text">
           XXX
       </span>
       <span class="right" data-type="formula">
           XXX
       </span>
    </div>
    <div data-type="step" data-id="2">
       <span class="left" data-type="text">
           XXX
       </span>
       <span class="right" data-type="formula">
           XXX
       </span>
    </div>
</section>

Drawing (DRW)

A question for which the answer must be drawn by the student. Examples includes questions that ask for a sketch or diagram. Questions will sometimes include the word “Draw” or “Label.” Note: graphs that cannot be developed using the JSXGraph tool will be considered DRW questions and an image should be developed for the required graph.

Example:

Draw a Punnett Square to show the genotypes that would result from a cross of two parents heterozygous for an allele (genotype Tt).

The correct response is:



  • A correct Punnett Square shows that the resulting genotypes (allele combinations) of the offspring are TT, Tt, and tt in the ratio 1:2:1.
  • To set up the Punnett Square, the parents' alleles are separated and placed across the top of the Punnett Square (for one parent) and repeated to the left of the Punnett Square (for the other parent).
  • Each cell in the Punnett Square is a possible allele combination for an offspring.

Directions for authoring DRW solutions:

When there is a single correct answer possible, the DRW solution should include two types of <section> blocks:

  • answer
  • explanation
<article>
    <header>
        ...
    </header>
    <main>
        <section class="answer" data-type="image">
            <div>
               <img src="images/XXX.svg">
            </div>
        </section>
        <section class="explanation" data-type="text">
            [text]
        </section>
    </main>
</article>

When the DRW question is open-ended and more than one correct response is possible, the DRW solution will include two types of <section> blocks:

  • sample response
  • explanation
<article>
    <header>
        ...
    </header>
    <main>
        <section class="sample_response" data-type="image">
            <div>
               <img src="images/XXX.svg">
            </div>
        </section>
        <section class="explanation" data-type="text">
            [text]
        </section>
    </main>
</article>

The first section, the answer or sample_response section will include the path to a single image (svg) file. The attributes for this section block are class="answer" or class="sample_response" and data-type="image".

<section class="answer" data-type="image">
    <div>
        <img src="images/XXX.svg">
    </div>
</section>

The second section, the explanation section, can be formatted as a bulleted list or step-by-step template to make the content more concise and easier to parse.

<section class="explanation" data-type="text">
    <p>
        The explanation can focus on a description of the drawing or provide guidance to the student on what should
        be included in their drawing (for open-ended questions).
    </p>
    <ul>
        <li>First point of guidance.</li>
        <li>Second point of guidance.</li>
        <li>Third point of guidance.</li>
    </ul>
</section>

Ledger (LDGR)

A question for which the answer is a ledger, journal, or accounting statement.

Example:

ABC Company creates a journal entry for June 1, 2018 detailing the company’s issuing of common stock (credit) in exchange for $100,000 in cash (debit). The cash account number is 1000, and the stock account number is 3000. Create a journal entry for the ABC Company.

Directions for authoring LDGR solutions:

The LDGR solution should include two types of <section> blocks:

  • answer
  • explanation

The second section, the explanation section, is commonly formatted as a step-by-step that details the steps taken to develop the ledger or statement (but is not required to be a step-by-step).

<article>
            <header>
                ...
            </header>
            <main>
                <section class="answer" data-type="image">
                    <div>
                       <img src="images/XXX.svg">
                    </div>
                </section>
                <section class="step_by_step" data-type="text">
                    [text]
                </section>
            </main>
        </article>

The first section, the answer section, will include the image (svg) file of the accounting statement. It will be created using the ledger templates provided in Google sheets.

<section class="answer" data-type="image">
            <div>
                <img src="images/XXX.svg">
            </div>
        </section>

The second section, the explanation section, is commonly formatted as a step-by-step that details the steps taken to develop the ledger or statement (but is not required to be a step-by-step).

<section class="step-by-step" data-type="text">
            <p>
                The step-by-step should provide detailed steps on how to create the accounting statement or ledger.
            </p>
        </section>

Matching (MAT)

A question where there is a set of givens and a set of choices and the question instructs you to match the givens with the choices in some way (by labelling, writing them in a space, etc.)

Example:

Match each organism with the best description.

____ cat
____ trout
____ snake
____ frog
a. amphibian
b. mammal
c. reptile
d. fish

The correct answer is:

b
d
c
a

Directions for authoring MAT solutions:

There are always two types of <section> blocks present in a MAT solution:

  • answer
  • explanation
<article>
    <header>
        ...
    </header>
    <main>
        <section class="answer" data-type="text" data-answer="1">b</section>
        <section class="answer" data-type="text" data-answer="2">d</section>
        <section class="answer" data-type="text" data-answer="3">c</section>
        <section class="answer" data-type="text" data-answer="4">a</section>
        <section class="explanation" data-type="text" data-answer="1">
               explanation for match to option b
        </section>
        <section class="explanation" data-type="text" data-answer="2">
                explanation for match to option d
        </section>
        <section class="explanation" data-type="text" data-answer="3">
               explanation for match to option c
        </section>
        <section class="explanation" data-type="text" data-answer="4">
               explanation for match to option a
        </section>
    </main>
</article>

For a MAT question, there will be multiple answer <section> blocks, the total depending on how many matches (often with corresponding blanks) are being made in the question stem. An answer <section> is provided for each match and numbered sequentially. The attributes for each answer section block are always class="answer", data-type="text", and data-answer="#" numbered sequentially to correspond with the blanks.

Example: 

<section class="answer" data-type="text" data-answer="1">[answer for first match]</section>
<section class="answer" data-type="text" data-answer="2">[answer for second match]</section>

For the next <section> blocks, an explanation for each match is provided to the student. The attributes for the section are class="explanation" and data-type="text". The data-answer attribute matches the sequence number assigned to that match in the answer sections.

Example: 

<section class="explanation" data-type="text" data-answer="1">
    A separate explanation is provided for each answer (e.g., for each match). The explanation and included content
    should be relevant to the specific match and help a student better understand the concept being tested.
</section>
<section class="explanation" data-type="text" data-answer="2">
    This would be a separate explanation for the second match.
</section>

Classifying (CL)

A question where a set of options are classified into a given group (usually two or more).

Example:

For the following organelles, classify them as organelles that are found in animal cells, plant cells, or both animal and plant cells.

a. cell wall
b. nucleus
c. central vacuole
d. ribosome
e. centriole

The correct answer is:

Animal cells: e
Plant cells: a, c
Both animal and plant cells: b, d

Directions for authoring CL solutions:

There are always three types of <section> blocks present in a CL solution:

  • group
  • answer
  • explanation
<article>
    <header>
        ...
    </header>
    <main>
        <section class="group" data-type="text" data-answer="1">animal cells</section>
        <section class="group" data-type="text" data-answer="2">plant cells</section>
        <section class="group" data-type="text" data-answer="3">both animal and plant cells</section>
        <section class="answer" data-type="text" data-answer="a">2</section>
        <section class="answer" data-type="text" data-answer="b">3</section>
        <section class="answer" data-type="text" data-answer="c">2</section>
        <section class="answer" data-type="text" data-answer="d">3</section>
        <section class="answer" data-type="text" data-answer="e">1</section>
        <section class="explanation" data-type="text" data-answer="1">
               explanation for all classifications into group "animal cells"
        </section>
        <section class="explanation" data-type="text" data-answer="2">
                explanation for all classifications into group "plant cells"
        </section>
        <section class="explanation" data-type="text" data-answer="3">
               explanation for all classifications into group "both plant and animal cells"
        </section>
    </main>
</article>

For a CL question, there will be multiple <section class="group"> blocks, the total depending on how many groups the question presents to categorize the options into.

Example: 

<section class="group" data-type="text" data-answer="1">[first group]</section>
<section class="group" data-type="text" data-answer="2">[second group]</section>
<section class="group" data-type="text" data-answer="3">[third group]</section>

For the next <section class="answer"> blocks, each option is classified into one of the possible groups.

Example: 

<section class="answer" data-type="text" data-answer="a">2</section>
<section class="answer" data-type="text" data-answer="b">3</section>
<section class="answer" data-type="text" data-answer="c">2</section>
<section class="answer" data-type="text" data-answer="d">3</section>
<section class="answer" data-type="text" data-answer="e">1</section>

For the next <section> blocks, an explanation for each group and all options classified into that group is provided to the student. The attributes for the section are class="explanation" and data-type="text". The data-answer attribute matches the group assigned to that number in the "group" sections.

Example: 

<section class="explanation" data-type="text" data-answer="1">
    A separate explanation is provided for each group (e.g., why these options are classified into this group).
</section>
<section class="explanation" data-type="text" data-answer="2">
    A separate explanation is provided for each group (e.g., why these options are classified into this group).
</section>
<section class="explanation" data-type="text" data-answer="3">
    A separate explanation is provided for each group (e.g., why these options are classified into this group).
</section>
